# Решение тестового кейса (SQL + бизнес-анализ)

## Описание
Кейс состоит из трёх частей:
1. **SQL** — написание запросов к базе данных опций и клиентов.
2. **План счетов банка** — определение номеров лицевых счетов по Положению Банка России № 809-П.
3. **Бизнес-процессы** — анализ и оптимизация процесса согласования договоров.

## Структура репозитория
- [`sql/`](sql/) — файл с SQL-запросами (`Задание 1.sql`).
- [`docs/`](docs/) — подробные пояснения к решению (файл `Пояснения.docx`).
- [`diagrams/`](diagrams/) — схемы AS IS (текущий процесс) и TO BE (целевой процесс) в нотации BPMN.

## Результаты
### SQL
- Все запросы отработаны корректно: использованы JOIN, агрегация, оконные функции, обработка NULL.
- Пояснения к каждому запросу даны в `docs/Пояснения.docx`.

### Банковские счета
- Номера счетов определены верно согласно нормативному документу.

### Бизнес-процессы
- Выявлены ключевые проблемы: многократные итерации, потеря версий, ручной учёт, отсутствие прозрачности.
- Предложены улучшения: переход на облачное хранилище (Google Docs / SharePoint), параллельное согласование, автоматическая генерация договоров.
- **Эффект:** время согласования сокращено с 5–7 дней до 1–2 дней.
